Creating VLANs on a Single & Multiple Switches

Lab: VLANs, trunking (802.1Q), and verifying traffic isolation with pings

In this lab, I used Cisco Packet Tracer to create and configure Virtual LANs (VLANs) on both a single switch and a two-switch network. By assigning hosts into different VLANs, configuring a trunk link, and testing with pings, I verified how VLANs split the network into separate broadcast domains and control which hosts can talk to each other.

Lab Goals

• Configure VLANs on a single switch and assign specific ports to each VLAN.
• Extend VLANs across two switches using an 802.1Q trunk link.
• Use pings to verify that devices in different VLANs cannot communicate directly.
• Observe how VLANs reduce broadcast domains and organize traffic.

Network 1 – Single Switch VLANs

  • Assigned IP addresses to each host on the first network.
  • Created VLANs (for example, VLAN 2 and VLAN 3) on the switch.
  • Moved specific switch ports into each VLAN.
  • Used ping tests to confirm that hosts in different VLANs could no longer reach each other.

Network 2 – VLANs Across Two Switches

  • Configured IP addresses for hosts on the second network and verified connectivity.
  • Created the same VLANs on the second switch and assigned the appropriate ports.
  • Tested pings between hosts in different VLANs to confirm separation.
  • Connected the two switches with a trunk link so VLAN tags could pass between them.

What I Learned

This lab showed me in a hands-on way that VLANs don’t just “label” ports—they actually create separate broadcast domains. When PCs were on the same VLAN, pings worked. As soon as I moved a PC into a different VLAN, pings failed unless routing or trunking was configured. Extending VLANs across two switches with a trunk link made it clear how bigger networks keep different departments separate but still organized on the same physical infrastructure.
